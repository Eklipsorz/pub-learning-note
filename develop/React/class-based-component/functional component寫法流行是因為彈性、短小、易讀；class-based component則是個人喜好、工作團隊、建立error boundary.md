## 描述

大部分場景都會因為以下幾點而開始流行起functional component寫法
1. 彈性較為足夠
2. 短小且易讀

這不表示functional component寫法會取代class-based component，而是通常會使用class-based的原因會是：
	- 個人喜好
	- 工作團隊共同使用
	- 建立 error boundary
